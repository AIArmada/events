# Package Boundary Addendum

This addendum now serves as a boundary reminder only.

## Current split

- Generic engagement behavior stays in `aiarmada/engagement`.
- Generic ticket and pass behavior stays in `aiarmada/ticketing`.
- Generic seat layout and allocation behavior stays in `aiarmada/seating`.
- `aiarmada/events` integrates those packages for event workflows but does not recreate their persistence or admin surfaces.
