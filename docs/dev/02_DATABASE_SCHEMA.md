# Database Schema Notes

This package no longer documents a standalone ticket or seating schema.

## Current ownership

- Event tables stay in `aiarmada/events`.
- Ticket and pass tables stay in `aiarmada/ticketing`.
- Seat map, hold, and allocation tables stay in `aiarmada/seating`.

When reviewing migrations, read the owning package directly instead of relying on this archived schema dump.
